⚛️ React Hooks – Practical Overview

useState STATE

Manages component state in functional components. Returns state value and setter function. Supports number, string, array, and object.

useEffect SIDE EFFECT

Handles API calls, subscriptions, and timers. Executes based on dependency array.

useEffect Dependencies LIFECYCLE

No deps → every render (avoid). Empty deps → once on mount. With deps → runs on change.

useContext GLOBAL STATE

Avoids props drilling by sharing data globally. Create → Provider → Consume.

Props vs Context COMPARISON

Props pass data manually through components. Context provides direct access anywhere.

useRef DOM

References DOM elements or mutable values. Updating ref does not re-render component.

useMemo PERFORMANCE

Memoizes expensive calculations. Improves performance by avoiding recalculation.

useCallback OPTIMIZATION

Memoizes functions. Prevents unnecessary re-renders in child components.

Callback Function FUNCTION

Function passed as an argument to another function. Used in events and component communication.

Rules of Hooks IMPORTANT

Call hooks only at top level. Use hooks only inside functional components.

Common Mistakes AVOID

Missing dependencies, overusing useEffect, unnecessary useMemo/useCallback.

Why Hooks Matter BEST PRACTICE

Hooks reduce boilerplate, improve readability, enable logic reuse, and simplify state management.
